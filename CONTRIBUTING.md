# Contributing to Vibeations

Thanks for wanting to contribute! Here's how.

## Quick Contributions

### Found a typo or broken link?
1. Fork the repo
2. Fix it
3. Open a PR

### Have a great resource to add?
1. Fork the repo
2. Add it to the appropriate section
3. Open a PR with a brief description

## Contribution Guidelines

### Adding Tools

Add to `docs/tools-comparison.md` with:
- Tool name and link
- Cost (free tier details)
- Best use case
- Your experience with it (optional)

### Adding Configs

Add to `configs/` with:
- Clear filename (e.g., `nextjs-supabase.cursorrules`)
- Comments explaining each section
- Tested and working

### Adding Cheatsheets

Add to `cheatsheets/` with:
- Markdown format
- Printable layout (fits on 1-2 pages)
- Most common commands first

## Style Guide

- **Be concise.** Less is more.
- **Be practical.** Real examples > theory.
- **Be honest.** Include limitations, not just benefits.
- **Be inclusive.** All skill levels welcome.

## Pull Request Process

1. Fork and create your branch (`git checkout -b feature/amazing-resource`)
2. Make your changes
3. Test any code examples
4. Commit with clear message (`git commit -m 'Add X resource'`)
5. Push to your branch (`git push origin feature/amazing-resource`)
6. Open a Pull Request

## Questions?

Open an issue or reach out on the Vibeations Discord.
